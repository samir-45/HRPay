SECTION 2 — ROLES & PERMISSIONS ENGINE (CRITICAL)
═══════════════════════════════════════
Build a fully dynamic, database-driven RBAC (Role-Based Access Control) system:

BUILT-IN ROLES (non-deletable):
  - Super Admin: full unrestricted access to everything across all tenants
  - HR Admin: full HR module access
  - Payroll Manager: payroll + finance modules
  - Department Manager: their own team only
  - Employee: self-service only

CUSTOM ROLES (Super Admin can create/edit/delete):
  - Super Admin can create any named custom role (e.g., "Regional HR", "Finance Auditor")
  - Each role has a granular permission matrix:
    * Per-module access: Dashboard, Employees, Payroll, Attendance, Leave, Recruitment, Performance, Training, Benefits, Reports, Settings, AI Assistant
    * Per-action access: View, Create, Edit, Delete, Export, Approve
    * Per-department scope: All departments, Own department only, Specific departments
  - Super Admin can assign any user to any role from any page (user card, employee profile, or dedicated Roles Management page)
  - Role changes take effect immediately without re-login
  - Permission inheritance and override rules
  - Audit log of all role/permission changes

ROLES MANAGEMENT PAGE:
  - Visual permission matrix grid (modules × actions as checkboxes)
  - Drag-and-drop role priority ordering
  - Role duplication / template system
  - See which users are assigned each role
  - Bulk role assignment

SECTION 21 — TECH STACK & PROJECT STRUCTURE
═══════════════════════════════════════
/app                    → Next.js App Router pages
/app/api               → API route handlers
/components            → Reusable UI components
/components/ai         → AI Assistant components
/lib                   → Prisma client, auth helpers, utils
/lib/ai                → AI provider abstraction layer (supports OpenAI SDK-compatible APIs)
/prisma                → schema.prisma + seed.ts
/public                → Static assets

Dependencies to install:
next, react, typescript, tailwindcss, @shadcn/ui, prisma, @prisma/client,
next-auth, bcryptjs, jsonwebtoken, zod (validation),
recharts (charts), react-pdf (payslip PDF), 
exceljs (Excel export), react-hook-form,
openai (works with OpenAI-compatible APIs including Groq, xAI),
@anthropic-ai/sdk, nodemailer, date-fns, lucide-react

AI Provider Abstraction Layer:
Create /lib/ai/providers.ts that routes to the correct SDK based on stored provider:
- OpenAI / Groq / xAI / Mistral → use openai SDK with custom baseURL
- Anthropic → use @anthropic-ai/sdk
- Google Gemini → use @google/generative-ai
All providers normalized to a single chat(messages, systemPrompt) function.
